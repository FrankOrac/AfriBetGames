## Introduction

AfriBet Games is a comprehensive gaming platform designed for African markets. The platform supports multiple game types, multi-currency transactions, and a vibrant community forum.

## Game Types

### Virtual Betting
- **Duration**: Runs for 35 virtual weeks
- **Number Range**: 0-40
- **Odds**: 1.01-7.00 (randomized per number)
- **Player Selection**: 5 numbers
- **Winning Condition**: 3 or more correct numbers
- **Timer**: 4 minutes 59 seconds countdown per round
- **Results**: Instant after each round

**How It Works**:
1. Each virtual week displays the current week number (e.g., "Week 12 of 35")
2. Countdown timer shows time until next draw
3. System automatically generates random winning numbers and odds
4. Results displayed immediately after countdown
5. Timer resets for next round

### Noon Game
- **Number Range**: 0-25
- **Odds**: 1.01-12.00 (randomized)
- **Player Selection**: 3 numbers
- **Bonus System**: 1 bonus number automatically generated by system
- **Winning Condition**: 2 or more correct numbers + bonus number
- **Result Time**: 12:00 PM daily

- **Number Range**: 0-25
- **Odds**: 1.01-12.00 (randomized)
- **Player Selection**: 3 numbers
- **Bonus System**: 1 bonus number automatically generated by system
- **Winning Condition**: 2 or more correct numbers + bonus number
- **Result Time**: 12:00 AM (midnight) daily

## Technical Architecture

### Frontend
- **Framework**: React 18 with TypeScript
- **Routing**: Wouter (lightweight routing)
- **State Management**: TanStack Query v5 for server state
- **Styling**: Tailwind CSS + Shadcn UI components
- **Animations**: Framer Motion
- **Forms**: React Hook Form with Zod validation

### Backend
- **Runtime**: Node.js with Express
- **Storage**: In-memory storage (MemStorage) for MVP
- **Type Safety**: Full TypeScript with Drizzle Zod schemas

### Key Features
1. **Dark Mode**: Full dark mode support with system preference detection
2. **Country Context**: Global country selection with currency formatting
3. **Theme Provider**: Centralized theme management
4. **Responsive Design**: Mobile-first approach

### Adding a New Game Type

1. **Update Schema** (`shared/schema.ts`):
```typescript
export enum GameType {
  // ... existing types
  NEW_GAME = 'new_game',
}
```

2. **Update Storage** (`server/storage.ts`):
```typescript
{
  type: GameType.NEW_GAME,
  name: 'New Game',
  description: 'Description...',
  minNumber: 0,
  maxNumber: 50,
  minOdds: 1.01,
  maxOdds: 10.00,
  numbersToSelect: 5,
  minMatches: 3,
  hasBonus: false,
}
```

3. **Create Game Page** (`client/src/pages/NewGame.tsx`)
4. **Register Route** in `client/src/App.tsx`

4. Click "Publish" button

## Best Practices

### Code Style
- Use TypeScript strict mode
- Follow existing component patterns
- Add data-testid attributes to interactive elements
- Use Shadcn components over custom components

### State Management
- Use TanStack Query for server state
- Use Context for global UI state (theme, country)
- Avoid prop drilling - use context when needed

### Styling
- Use Tailwind utility classes
- Follow dark mode patterns with `dark:` variants
- Use CSS variables for theme colors
- Keep responsive design mobile-first

### Performance
- Lazy load routes and heavy components
- Use React Query caching effectively
- Optimize images and assets
- Minimize bundle size
